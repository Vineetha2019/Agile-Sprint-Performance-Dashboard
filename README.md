# Agile Sprint Performance Dashboard
This project analyzes Agile sprint performance using key delivery metrics to evaluate team efficiency, workflow stability, and sprint execution patterns.
## Problem Statment
To analyze the sprint performance of a development team using Agile methodology and identify improvement areas through data-driven insights.
## Analyzed Metrics
 - Sprint Velocity with average Trend
 - Cycle time trend analysis
 - Sprint Thoughput
 - Burndown chart for one sprint
## Insights
- Velocity varies due to task complexity differences 
- Cycle time shows workflow inconsistency
- Throughput remains stable across sprints
- Burndown shows late sprint completion trend
- Sprint execution is stable but unevenly distributed
## Recommendations
 - Standardize task sizing for consistent velocity based on complexity, risk and impact
 - Improve task breakdown to stabilize cycle time 
 - Distribute workload evenly across sprint 
 - Start working on tasks as soon as story is ready 
 - Track progress mid-sprint to avoid end loading or include in daily scrum meeting 
## Tools & Techniques Used
- Microsoft Excel 
- Agile Metrics Analysis  
- Data Visualization (Charts & Dashboards)
## Business Impact
This analysis helps teams and organizations to:
- Improve Agile delivery efficiency  
- Reduce workflow bottlenecks  
- Enhance sprint planning accuracy  
- Achieve more predictable delivery cycles  
- Support data-driven decision-making
